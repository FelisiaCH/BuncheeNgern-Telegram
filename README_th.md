# FinTrack

[Read the English guide here](README.md)

PWA สำหรับบันทึกรายรับ-รายจ่ายแบบรองรับหลายสาขา ออกแบบมาให้ใช้งานบนมือถือเป็นหลัก พนักงานเข้าสู่ระบบด้วย Google บันทึกรายการ (สามารถแยกเป็นหลายสกุลเงินในรายการเดียวได้) แล้วข้อมูลจะถูกเพิ่มเข้า Google Sheet — รูปสลิปจะถูกอัปโหลดไปยัง Drive และข้อความแจ้งเตือนจะถูกส่งไปยัง Telegram ของผู้ดูแล ฝั่งหน้าเว็บเป็นไฟล์เดียว `index.html` ส่วนฝั่งเซิร์ฟเวอร์เป็นไฟล์ Google Apps Script เดียว (`Code.gs`)

## ฟีเจอร์

- บันทึกแยกตามสาขา จัดการสาขาได้จากแท็บ ตั้งค่า ในแอป
- หลายสกุลเงินในรายการเดียว — เพิ่มแถวสกุลเงิน/จำนวนเงินได้หลายแถวในรายการเดียวกัน (เช่น จ่ายส่วนหนึ่งเป็น LAK อีกส่วนเป็น THB) ทุกแถวจะมี Transaction ID เดียวกัน
- แจ้งเตือนผ่าน Telegram ไปยังแชทส่วนตัว กลุ่ม หรือทั้งสองอย่างพร้อมกัน
- รองรับ 5 ภาษา: ลาว, ไทย, อังกฤษ, เวียดนาม, พม่า
- เป็น PWA ติดตั้งลงเครื่องได้ และใช้งาน app shell แบบออฟไลน์ได้

> **ข้อควรรู้เรื่องความปลอดภัย:** การล็อกอินในแอปนี้เป็นเพียง *การกรองเบื้องต้น* ไม่ใช่ระบบควบคุมสิทธิ์การเข้าถึงจริง โทเค็น Google (JWT) จะถูกอ่านที่ฝั่งเบราว์เซอร์เท่านั้น และไม่มีการตรวจสอบที่ฝั่งเซิร์ฟเวอร์ และ Apps Script backend ถูกดีพลอยแบบ "Who has access: Anyone" (จำเป็นเพื่อไม่ให้ Web App รีไดเรกต์ไปหน้าล็อกอิน) หน้าล็อกอินมีไว้เพื่อบันทึกว่า "ใครเป็นคนบันทึกรายการนี้" สำหรับทีมภายในที่ไว้ใจกันได้ — ใครก็ตามที่มี URL ของ Web App สามารถเรียกใช้งานได้โดยตรง อย่าใช้สิ่งนี้เป็นตัวควบคุมการเข้าถึงข้อมูลที่สำคัญ ดูหัวข้อ [การเสริมความปลอดภัย](#การเสริมความปลอดภัย-ไม่บังคับ) หากต้องการเพิ่มความปลอดภัย

## สิ่งที่ต้องมีก่อนเริ่ม

- บัญชี Google (สำหรับ Sheet, โฟลเดอร์ Drive, โปรเจกต์ Apps Script และ OAuth client)
- Telegram bot token (จาก @BotFather) และ chat ID อย่างน้อยหนึ่งรายการสำหรับรับการแจ้งเตือน
- พื้นที่สำหรับโฮสต์ทั้งโปรเจกต์บน **HTTPS** (เช่น Cloudflare Pages, Netlify, GitHub Pages) Google Sign-In **ใช้งานไม่ได้** กับ `file://` หรือ `http://` ธรรมดา

มีค่า placeholder ที่ต้องกรอกทั้งหมด **6 ค่า** ก่อนแอปจะใช้งานได้:

| ไฟล์ | ค่าคงที่ | คืออะไร |
| --- | --- | --- |
| `Code.gs` | `SPREADSHEET_ID` | ID ของ Google Sheet (จาก URL) |
| `Code.gs` | `DRIVE_FOLDER_ID` | ID ของโฟลเดอร์ Drive สำหรับเก็บรูปสลิป |
| `Code.gs` | `TELEGRAM_BOT_TOKEN` | Token ของบอทจาก @BotFather |
| `Code.gs` | `TELEGRAM_CHAT_IDS` | **อาร์เรย์** ของ chat ID ที่จะรับการแจ้งเตือน — หนึ่งรายการหรือมากกว่า ทั้งแชทส่วนตัวและ/หรือกลุ่ม |
| `index.html` | `GOOGLE_CLIENT_ID` | OAuth Web client ID (สำหรับ Google Sign-In) |
| `index.html` | `SCRIPT_URL` | URL `/exec` ของ Apps Script Web App ที่ดีพลอยแล้ว |

## 1. Backend — Google Sheets + Apps Script

1. สร้าง (หรือเปิด) Google Sheet แล้วคัดลอก ID จาก URL: `docs.google.com/spreadsheets/d/`**`ส่วนนี้`**`/edit`
2. สร้างโฟลเดอร์ Drive สำหรับเก็บรูปสลิป แล้วคัดลอก ID จาก URL
3. ในชีต ไปที่ **Extensions ▸ Apps Script** ลบโค้ดเดิมทั้งหมด แล้ววางเนื้อหาทั้งหมดของ `Code.gs`
4. สร้างไฟล์ `appsscript.json` ในโปรเจกต์เดียวกัน (ใน **Project Settings** ต้องเปิด "Show `appsscript.json` manifest file" ก่อน) แล้ววางเนื้อหาของ `appsscript.json` จากรีโปนี้ ซึ่งประกาศ scope ที่สคริปต์ต้องใช้:
   - `https://www.googleapis.com/auth/spreadsheets`
   - `https://www.googleapis.com/auth/drive`
   - `https://www.googleapis.com/auth/script.external_request` (จำเป็นสำหรับการเรียก Telegram API)
5. กรอกค่าคงที่ด้านบนของ `Code.gs`:

   ```js
   const SPREADSHEET_ID  = 'YOUR_GOOGLE_SHEET_ID_HERE';
   const DRIVE_FOLDER_ID = 'YOUR_DRIVE_FOLDER_ID_HERE';
   const TELEGRAM_BOT_TOKEN = 'YOUR_TELEGRAM_BOT_TOKEN_HERE';
   const TELEGRAM_CHAT_IDS  = ['YOUR_TELEGRAM_CHAT_ID_HERE'];
   ```

6. **ขออนุญาตสคริปต์ (Authorize)** ในตัวแก้ไข เลือกฟังก์ชันใดก็ได้จากดรอปดาวน์ (เช่น `todayTab`) แล้วกด **Run** หนึ่งครั้ง Google จะแสดงหน้ายินยอม — กดยืนยัน การยินยอมครั้งเดียวนี้ครอบคลุม scope ทั้งหมดที่ประกาศใน `appsscript.json` รวมถึงสิทธิ์ external request (สำหรับเรียก Telegram) และสิทธิ์ Drive (สำหรับอัปโหลดสลิป) **หากข้ามขั้นตอนนี้ รายการจะยังบันทึกเข้าชีตได้ แต่ข้อความ Telegram และการอัปโหลดสลิปจะล้มเหลวแบบเงียบ ๆ**
7. กด **Deploy ▸ New deployment ▸ Web app**:
   - **Execute as:** Me
   - **Who has access:** Anyone
8. คัดลอก **Web app URL** (ลงท้ายด้วย `/exec`) เพื่อนำไปวางใน `index.html` เป็นค่า `SCRIPT_URL`

> **ทุกครั้งที่แก้ไข `Code.gs` (หรือ `appsscript.json`) ต้องดีพลอยเวอร์ชันใหม่** — ไปที่ **Deploy ▸ Manage deployments ▸ edit (ไอคอนดินสอ) ▸ Version: New version ▸ Deploy** การแก้โค้ดเพียงอย่างเดียวจะ *ไม่* อัปเดตการทำงานของ URL `/exec` ที่ใช้งานอยู่

### สิ่งที่ backend เปิดให้ใช้งาน

- `doGet`: `?action=getTodayData` (รายการของวันนี้) และ `?action=getDateData&date=DD-MM-YYYY` (รายการของวันที่ระบุ)
- `doPost`: `{ action: 'submitEntry', ... }` — อัปโหลดสลิป (ถ้ามี) ไปยัง Drive, เพิ่มแถวลงชีตของวันนั้นหนึ่งแถวต่อหนึ่งสกุลเงิน (ทุกแถวมี `Transaction ID` เดียวกัน) และส่งข้อความแจ้งเตือนผ่าน Telegram
- ชีตของแต่ละวันจะถูกสร้างขึ้นโดยอัตโนมัติเมื่อใช้งานครั้งแรก พร้อมหัวคอลัมน์: `Timestamp, Staff Name, Item Name, Currency, Price, Type, Shop, Payment Method, Slip URL, Transaction ID`

## 2. Telegram bot

1. ในเทเลแกรม ส่งข้อความถึง **@BotFather** พิมพ์ `/newbot` ทำตามขั้นตอน แล้วคัดลอก **bot token** — นั่นคือ `TELEGRAM_BOT_TOKEN`
2. หา chat ID ที่ต้องการให้รับการแจ้งเตือน:
   - **แชทส่วนตัว (ตัวเองหรือพนักงาน):** บอทไม่สามารถส่งข้อความถึงคุณก่อนได้ ต้องส่งข้อความถึงบอทก่อน จากนั้นเปิด `https://api.telegram.org/bot<YOUR_TOKEN>/getUpdates` ในเบราว์เซอร์แล้วดูค่า `message.chat.id` — เป็นตัวเลขบวกธรรมดา
   - **แชทกลุ่ม:** เพิ่มบอทเข้ากลุ่ม ส่งข้อความในกลุ่มสักครั้ง แล้วตรวจ `getUpdates` เพื่อดู `chat.id` ของ update นั้น — ID ของกลุ่มจะเป็นเลขลบ มักขึ้นต้นด้วย `-100`
3. นำ ID หนึ่งรายการหรือมากกว่าไปใส่ในอาร์เรย์ จะเป็นสตริงหรือตัวเลขก็ได้:

   ```js
   const TELEGRAM_CHAT_IDS = ['123456789', '-1009876543210'];
   ```

   ทุกรายการที่ไม่ใช่ placeholder จะได้รับสำเนาของข้อความแจ้งเตือน หากต้องการแจ้งเตือนไปที่เดียว ให้คงค่า `['YOUR_TELEGRAM_CHAT_ID_HERE']` ไว้ (หรือลบรายการที่ไม่ใช้ออก)

> ข้อผิดพลาด "chat not found" มักเกิดจากการที่ยังไม่เคยส่งข้อความถึงบอทก่อน (สำหรับแชทส่วนตัว) หรือบอทไม่ได้อยู่ในกลุ่ม หรือ ID ผิด/ขาดเครื่องหมาย `-` นำหน้า

## 3. Google Sign-In — OAuth client

จำเป็นต้องตั้งค่านี้ก่อน ปุ่มเข้าสู่ระบบจึงจะปรากฏ

1. ไปที่ **console.cloud.google.com ▸ APIs & Services ▸ Credentials**
2. **Create Credentials ▸ OAuth client ID ▸ Application type: Web application**
3. ในช่อง **Authorized JavaScript origins** ให้เพิ่ม origin ที่ตรงกับที่โฮสต์แอป เช่น `https://yourproject.pages.dev` (เฉพาะ scheme + host **เท่านั้น** ไม่มี path ไม่มี trailing slash และต้องเป็น `https://` ห้ามเป็น `file://`) เพิ่มทุก origin ที่จะใช้เปิดแอปจริง รวมถึง URL สำหรับทดสอบ/พรีวิวด้วย
4. คัดลอก **Client ID** — นั่นคือ `GOOGLE_CLIENT_ID`

> หากปุ่มเข้าสู่ระบบปรากฏแต่กดแล้วไม่เกิดอะไรขึ้น ให้เปิด console ของเบราว์เซอร์ — ข้อความ `origin is not allowed for the given client ID` แปลว่า origin ในขั้นตอนที่ 3 ไม่ตรงกับ URL ที่ใช้เปิดหน้าเว็บ (รวมถึง scheme ด้วย)

หาก `GOOGLE_CLIENT_ID` ยังเป็นค่า placeholder อยู่ หรือสคริปต์ Google Identity Services โหลดไม่สำเร็จ หน้าล็อกอินจะแสดงคำแนะนำอธิบายปัญหาแทนการแสดงพื้นที่เปล่า

## 4. ตั้งค่าฝั่งหน้าเว็บ + การโฮสต์

1. ใกล้ด้านบนของบล็อก `<script>` ใน `index.html` ให้ตั้งค่าคงที่สองตัว:

   ```js
   const GOOGLE_CLIENT_ID = "…OAuth client ID ของคุณ…";
   const SCRIPT_URL       = "…URL /exec ของ Apps Script…";
   ```

   ค่าทั้งสองนี้เป็นค่าคงที่ในไฟล์ ไม่มีช่องตั้งค่าในแอป
2. ดีพลอย **ทั้งโปรเจกต์** ไม่ใช่แค่ `index.html` — แอปโหลด `i18n/languages.js`, `service-worker.js` และไฟล์ไอคอน/manifest จาก path สัมพัทธ์ ถ้าไฟล์เหล่านี้ 404 แอปจะโหลดไม่สำเร็จหรือไม่มีคำแปล จากโฟลเดอร์หลักของรีโป:

   ```bash
   npx wrangler pages deploy .
   ```

   (หรือวิธี "deploy โฟลเดอร์นี้" ที่เทียบเท่าบน Netlify/GitHub Pages) การดีพลอยแค่ `index.html` เป็นข้อผิดพลาดที่พบบ่อยและจะทำให้แอปใช้งานไม่ได้
3. เปิดหน้าเว็บผ่าน URL ที่เป็น HTTPS
4. **bump เวอร์ชัน cache ของ service worker ทุกครั้งที่มีการเปลี่ยนแปลง** ใน `service-worker.js` มีบรรทัด:

   ```js
   const CACHE = 'fintrack-v1.1.2';
   ```

   หน้า HTML จะถูกโหลดแบบ network-first (ดังนั้นการเปลี่ยนแปลงส่วนใหญ่ใน `index.html` จะถึงผู้ใช้เมื่อรีโหลดหน้าตามปกติ) แต่ `i18n/languages.js`, ไอคอน และไฟล์ static อื่น ๆ จะถูกโหลดแบบ cache-first เมื่อเปลี่ยนไฟล์ static ใด ๆ ให้ bump ค่า `CACHE` (เช่น `v1.1.3`) เพื่อล้างไฟล์เก่าออกจาก cache และโหลดไฟล์ใหม่

## 5. คู่มือการใช้งาน

- **บันทึกรายการ:** ที่แท็บ **บันทึก** เลือก รายรับ หรือ รายจ่าย กรอกชื่อรายการ (พิมพ์ใหม่หรือเลือกจากรายการที่ใช้ก่อนหน้านี้) กรอกจำนวนเงินและสกุลเงิน เลือกสาขาและช่องทางชำระเงิน
- **หลายสกุลเงินในรายการเดียว:** กด **"+ เพิ่มสกุลเงิน"** เพื่อเพิ่มแถวสกุลเงิน/จำนวนเงิน (เช่น จ่ายส่วนหนึ่งเป็น LAK อีกส่วนเป็น THB) แต่ละแถวจะถูกบันทึกเป็นแถวแยกในชีต แต่ทุกแถวจากการบันทึกครั้งเดียวจะมี Transaction ID เดียวกัน เพื่อให้จัดกลุ่มดูได้ในภายหลัง
- **แนบสลิป:** หากเลือกช่องทางชำระเงินเป็น **Lao QR / OnePay** จะมีพื้นที่อัปโหลดปรากฏขึ้น — แตะเพื่อเลือกไฟล์ หรือลากรูปมาวาง (ขนาดไม่เกิน 5 MB) รูปจะถูกย่อขนาดให้ด้านที่ยาวที่สุดไม่เกิน 1280px และแปลงเป็น JPEG คุณภาพประมาณ 82% ที่ฝั่งเบราว์เซอร์โดยอัตโนมัติก่อนอัปโหลด เพื่อให้ไฟล์มีขนาดเล็ก
- **เปลี่ยนสาขา:** เลือกชิปสาขาในแท็บบันทึก จัดการสาขา (เพิ่ม/ลบ) ได้จาก **ตั้งค่า ▸ Management**
- **เปลี่ยนวันที่ / ดูแดชบอร์ด:** ที่แท็บ **สรุป** ใช้ปุ่มลูกศรเพื่อเลื่อนไปวันก่อนหน้า/ถัดไป กรองตามสาขา และสลับแท็บสกุลเงิน (LAK/THB/USD) เพื่อดูรายรับเงินสด รายรับ QR/OnePay และรายจ่ายรวมของวันนั้น พร้อมรายการล่าสุด
- **เปลี่ยนภาษา:** ในหน้า **ตั้งค่า** เลือกได้ระหว่างลาว, ไทย, อังกฤษ, เวียดนาม หรือพม่า — ระบบจะจดจำค่าที่เลือกไว้บนอุปกรณ์

## การเสริมความปลอดภัย (ไม่บังคับ)

หากต้องการให้การล็อกอินเป็นระบบควบคุมสิทธิ์การเข้าถึงจริง:

- ส่งโทเค็น Google (JWT) ไปยัง backend ทุกครั้งที่บันทึกข้อมูล แล้วตรวจสอบที่ฝั่งเซิร์ฟเวอร์ — ตรวจ signature, ตรวจว่า `aud` ตรงกับ `GOOGLE_CLIENT_ID` ของคุณ และตรวจว่าอีเมลอยู่ใน allow-list — ก่อนเพิ่มแถวลงชีต
- คงค่า "Who has access: Anyone" ไว้ (Apps Script ต้องใช้ค่านี้เพื่อไม่ให้รีไดเรกต์ไปหน้าล็อกอิน) แต่ให้ `doPost` ปฏิเสธคำขอที่ไม่มีโทเค็นหรือตรวจสอบไม่ผ่าน
